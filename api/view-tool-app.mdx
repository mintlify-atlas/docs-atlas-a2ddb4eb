## Overview

The `ViewToolApp` class extends `javax.swing.JFrame` and serves as the main application window for ViewTool. It manages the overall layout, navigation between panels, user authentication state, and the media synchronization polling component.

<Note>
This is the entry point of the ViewTool Swing application. All panels are displayed within this frame's dynamic panel container.
</Note>

## Constructor

### ViewToolApp()

```java
public ViewToolApp()
```

Initializes the main application frame. This constructor performs the following:

1. Applies global UI styling via `applyGlobalUI()`
2. Initializes Swing components
3. Builds the layout structure
4. Retrieves stored authentication token
5. Initializes all panels (PanelMain, PanelManagement, PanelPreferences)
6. Shows Login panel if no token exists, otherwise calls `loggedSuccess()`

<CodeGroup>
```java Example Usage
ViewToolApp app = new ViewToolApp();
app.setVisible(true);
```
</CodeGroup>

### buildLayout()

```java
private void buildLayout()
```

Constructs the main window layout using BorderLayout:
- **NORTH**: Navigation buttons (Home and Settings)
- **CENTER**: Dynamic panel container for switching between views

### loggedSuccess()

<ParamField path="loggedSuccess" type="void">
  Called after successful authentication to display the main application interface
</ParamField>

```java
public final void loggedSuccess(String token)
```

**Parameters:**
- `token` (String): Authentication token for the logged-in user

**Actions:**
1. Stores the authentication token
2. Sets `loggedIn` to true
3. Displays PanelMain and PanelManagement in split view
4. Makes Settings button visible

<CodeGroup>
```java Example
String authToken = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...";
app.loggedSuccess(authToken);
```
</CodeGroup>

### showPanel()

```java
private void showPanel(JPanel panel)
```

**Parameters:**
- `panel` (JPanel): The panel to display in the dynamic container

Replaces the current panel with the specified panel. This method:
- Clears the dynamic panel container
- Adds the new panel to CENTER position
- Toggles navigation button visibility based on login state
- Revalidates and repaints the UI
- Centers the window on screen

<Note>
This method is used for single-panel views like Login or Preferences.
</Note>

### showMainAndManagement()

```java
public void showMainAndManagement(JPanel top, JPanel bottom)
```

**Parameters:**
- `top` (JPanel): Panel to display at the top (typically PanelMain)
- `bottom` (JPanel): Panel to display at the bottom (typically PanelManagement)

Creates a combined view with two panels stacked vertically:
- Top panel has fixed height of 167 pixels
- Bottom panel fills remaining space
- Hides Home button, shows Settings button

<CodeGroup>
```java Usage in Constructor
showMainAndManagement(panelMain, panelManagement);
```
</CodeGroup>

### MenuLogoutItemActionPerformed()

```java
private void MenuLogoutItemActionPerformed(ActionEvent evt)
```

Handles user logout:
1. Shows Login panel
2. Sets `loggedIn` to false
3. Deletes stored token
4. Stops media polling
5. Hides navigation buttons

<CodeGroup>
```java Logout Flow
if (isLoggedIn()) {
    showPanel(new Login(this));
    setLoggedIn(false);
    tokenService.deleteToken();
    mediaSyncPolling.setRunning(false);
    panelButtonMainAndSetting.setVisible(false);
}
```
</CodeGroup>
