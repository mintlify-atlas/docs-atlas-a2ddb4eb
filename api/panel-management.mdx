## Overview

The `PanelManagement` class extends `JPanel` and provides a comprehensive interface for managing both local downloaded media files and network-based media library. It features a local file list, a paginated network media table, search functionality, and media operations.

<Note>
This is the most complex UI component in ViewTool, featuring pagination, live search, background polling, dual data sources (local + network), and a sophisticated loading overlay system.
</Note>

### initialLoad()

```java
private void initialLoad()
```

Uses a `Timer` with 0 delay to load data after panel is painted:
1. Sets initial load flags
2. Freezes UI
3. Refreshes local file list
4. Refreshes table data
5. Starts polling for network updates

<CodeGroup>
```java Load Sequence
Timer timer = new Timer(0, new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        initialLoadInProgress = true;
        localReady = false;
        networkReady = false;
        freezeUI();
        refreshLocalList();
        refreshTableData(true);
        startPolling();
    }
});
timer.setRepeats(false);
timer.start();
```
</CodeGroup>

## Network Media Management

### startPolling()

```java
private void startPolling()
```

Activates the MediaSyncPolling component to periodically check for new network media:
1. Gets MediaSyncPolling from parent frame
2. Sets authentication token
3. Sets initial polling interval to 5 seconds
4. Attaches custom event listener (only once)
5. Starts polling
6. Changes interval to 60 seconds after first load

<CodeGroup>
```java Polling Configuration
MediaSyncPolling mediaSyncPolling = jframe.getComponent();
mediaSyncPolling.setToken(token);
mediaSyncPolling.setPollingInterval(5); // Initial
mediaSyncPolling.setRunning(true);
mediaSyncPolling.setPollingInterval(60); // Ongoing
```
</CodeGroup>

**Event Listener:**
When `CheckListMediaEvent` is received:
1. Extracts new media list
2. Sets `networkReady` to true on initial load
3. Calls `applyNetworkUpdateTable()` to update table
4. Checks synchronization

### refreshTableData()

```java
private void refreshTableData(boolean isInitialLoad)
```

**Parameters:**
- `isInitialLoad` (boolean): Whether this is the initial data load

Refreshes the media table in background:
1. Cancels any existing rebuild worker
2. Freezes UI if initial load
3. Creates new SwingWorker
4. Background: Calls `mediaService.createMediaListCombined()`
5. Done: Updates `fullMediaList` and applies to table model
6. Sets `localReady` and checks synchronization

<CodeGroup>
```java Background Rebuild
rebuildWorker = new SwingWorker<>() {
    @Override
    protected List<Media> doInBackground() {
        return mediaService.createMediaListCombined();
    }

    @Override
    protected void done() {
        if (isCancelled()) return;
        fullMediaList = get();
        applyMediaTableModel(fullMediaList);
        if (isInitialLoad) {
            localReady = true;
            checkSynchronization();
        }
    }
};
rebuildWorker.execute();
```
</CodeGroup>

### applyMediaTableModel()


## Search and Filtering

### LiveSearchOnTable()

```java
private void LiveSearchOnTable()
```

Attaches a `DocumentListener` to the search text field for real-time filtering:
- Listens to: `insertUpdate`, `removeUpdate`, `changedUpdate`
- Calls `applyTableFilter()` on any text change
- Sets `isLiveSearchActive` flag to prevent duplicate listeners

<Note>
Search is applied as the user types, providing instant feedback without requiring a search button.
</Note>

### applyTableFilter()

```java
private void applyTableFilter()
```

Filters the media table based on search text:
1. Gets search text from text field (lowercase)
2. If empty: Displays full media list
3. If not empty: Filters by matching filename, nickname, or URL
4. Updates table model with filtered list
5. Resets to page 1
6. Updates pagination controls

<CodeGroup>
```java Filter Logic
for (Media media : fullMediaList) {
    String fileName = (media.mediaFileName != null) ? media.mediaFileName.toLowerCase() : "";
    String nickName = tableModel.getUserNames().getOrDefault(media.userId, "").toLowerCase();
    String url = (media.downloadedFromUrl != null) ? media.downloadedFromUrl.toLowerCase() : "";

    if (fileName.contains(searchText) || nickName.contains(searchText) || url.contains(searchText)) {
        filtered.add(media);
    }
}
```
</CodeGroup>

## Pagination

### updatePagination()

```java
private void updatePagination()
```

Updates pagination controls and display:
1. Calculates total pages: `Math.ceil(totalItems / pageSize)`
2. Updates page status label (e.g., "Page 2 of 5")
3. Enables/disables Prev/Next buttons based on current page
4. Updates table model to show current page
5. Scrolls table to top

<CodeGroup>
```java Page Calculation
int totalItems = fullMediaList.size();
double result = totalItems / pageSize;
int totalPages = (int) Math.ceil(result);

labelPageStatus.setText("Page " + currentPage + " of " + totalPages);
buttonNext.setEnabled(currentPage < totalPages);
buttonPrev.setEnabled(currentPage > 1);
```
</CodeGroup>

### buttonOpenFileActionPerformed()

```java
private void buttonOpenFileActionPerformed(ActionEvent evt)
```

Opens the selected media file locally:
1. Gets selected media
2. Creates SwingWorker to open file in background
3. Calls `managementService.openLocalFile()`
4. Shows "Opening..." feedback on success
5. Shows "This media is not downloaded." on error

<CodeGroup>
```java Open File
SwingWorker<Void, Void> openFileWorker = new SwingWorker<>() {
    @Override
    protected Void doInBackground() throws Exception {
        managementService.openLocalFile(file);
        return null;
    }

    @Override
    protected void done() {
        try {
            get();
            utils.showFeedback(infoLabel, "Opening...", false);
        } catch (InterruptedException | ExecutionException ex) {
            utils.showFeedback(infoLabel, "This media is not downloaded.", true);
        }
    }
};
openFileWorker.execute();
```
</CodeGroup>

### buttonUploadFileActionPerformed()

```java
private void buttonUploadFileActionPerformed(ActionEvent evt)
```

Uploads a local file to the network library:
1. Opens JFileChooser for file selection
2. Prompts user for YouTube URL in infinite loop:
   - Allows cancel
   - Rejects empty URLs
3. Calls `managementService.uploadMedia()`
4. Shows success alert with file path
5. Shows error feedback on failure

<CodeGroup>
```java URL Prompt Loop
while (true) {
    youtubeUrl = JOptionPane.showInputDialog(this, "Enter the YouTube URL:");
    if (youtubeUrl == null) break; // User cancelled
    if (!youtubeUrl.trim().isEmpty()) break; // Valid URL
    utils.showFeedback(infoLabel, "URL can't be empty.", true);
}
```
</CodeGroup>
