## Overview

The `PanelMain` class extends `JPanel` and provides the main download interface for ViewTool. Users can enter a URL, select format (MP3/MP4), choose audio-only mode, and download media content to their local directory.

<Note>
This panel integrates with yt-dlp for downloading and uses SwingWorker for background processing to keep the UI responsive.
</Note>

- **Row 0**: YouTube icon, URL label, text field, "Only Audio" checkbox

## Core Functionality

### handleDownload()

```java
private void handleDownload()
```

Executes the download process in a background thread using `SwingWorker<Integer, Integer>`:

**Process Flow:**
1. Disables download button
2. Resets progress bar to 0
3. Validates URL and speed limit
4. Shows info alert: "The download is starting, please wait..."
5. Starts download with selected format and options
6. Updates progress bar as download proceeds
7. On completion: shows success message or error feedback
8. Re-enables download button

<CodeGroup>
```java Download Parameters
controller.startDownload(
    textFieldURL.getText(),
    buttonMP3.isSelected() ? MediaFormat.MP3 : MediaFormat.MP4,
    checkBoxOnlyAudio.isSelected(),
    prefService.getOutputDir(),
    log,
    pct -> publish(pct), // Progress consumer
    limit,
    preferences.isM3USelected()
);
```
</CodeGroup>

**Progress Updates:**
- Uses `publish()` to send progress percentage from background thread
- `process()` method updates the progress bar on EDT
- Switches from indeterminate to determinate when first percentage received

**Error Handling:**
- Invalid URL: Shows "Invalid URL." in info label
- Download failure: Shows "Downloaded uncompleted. Try again."
- Success: Shows "Download complete to [output directory]"

<Note>
The download operation uses yt-dlp under the hood. Make sure yt-dlp is installed and the path is configured in preferences.
</Note>

### handlePlayVideo()

```java
private void handlePlayVideo()
```

Opens the last downloaded file in a background thread using `SwingWorker<Void, Void>`:

**Process Flow:**
1. Calls `playService.playLastDownloaded()` with output directory
2. On success: Shows "Opening..." feedback
3. On error: Shows "You don't have videos yet." feedback

<CodeGroup>
```java Play Last Video
SwingWorker<Void, Void> openFileWorker = new SwingWorker<>() {
    @Override
    protected Void doInBackground() throws Exception {
        playService.playLastDownloaded(prefService.getOutputDir());
        return null;
    }

    @Override
    protected void done() {
        try {
            get();
            utils.showFeedback(infoLabel, "Opening...", false);
        } catch (InterruptedException | ExecutionException ex) {
            utils.showFeedback(infoLabel, "You don't have videos yet.", true);
        }
    }
};
openFileWorker.execute();
```
</CodeGroup>

## Threading Model

The panel uses proper threading to maintain UI responsiveness:

<Note>
**Important**: All long-running operations (download, file opening) run in SwingWorker background threads, never blocking the Event Dispatch Thread (EDT).
</Note>

### Download Thread
- Type: `SwingWorker<Integer, Integer>`
- Background task: `doInBackground()` runs download
- Progress updates: `process()` updates UI
- Completion: `done()` shows final message

### Play Thread
- Type: `SwingWorker<Void, Void>`
- Background task: Opens file with system player
- Completion: Shows success or error feedback
