## Overview

The `Login` class (Login.java:32) provides the authentication interface for ViewTool. It features a clean, centered form layout with email and password fields, a "remember me" checkbox, and automatic token management.

## Layout Architecture

The login panel uses a BorderLayout with a centered GridBagLayout form:

```java
public Login(ViewToolApp jframe) {
    this.tokenService = jframe.getTokenService();
    setLayout(new BorderLayout());
    setName("Login");
    setBorder(new EmptyBorder(20, 20, 20, 20));
    
    JPanel formPanel = new JPanel(new GridBagLayout());
    GridBagConstraints gbc = new GridBagConstraints();
    gbc.insets = new Insets(5, 5, 5, 5);
    gbc.fill = GridBagConstraints.HORIZONTAL;
    
    add(formPanel, BorderLayout.CENTER);
}
```

## Authentication Flow

### Login Process

When the user clicks the Login button:

1. **Input Validation**: Check for empty fields
2. **API Authentication**: Call MediaSyncPolling.login()
3. **Token Management**: Store token if authentication succeeds
4. **Session State**: Update logged-in status
5. **Token Persistence**: Save token if "Remember me" is checked
6. **Navigation**: Redirect to main interface

```java
buttonLogin.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        String email = fieldEmail.getText();
        String password = new String(fieldPassword.getPassword());
        boolean remember = checkBoxRemember.isSelected();
        
        MediaSyncPolling mediaSyncPolling = jframe.getComponent();
        
        if (email.isBlank() || password.isBlank()) {
            utils.showFeedback(infoLabel, "Can't be empty.", true);
            return;
        }
        
        try {
            token = mediaSyncPolling.login(email, password);
        } catch (Exception ex) {
            utils.showFeedback(infoLabel, 
                "Email or password not found.", true);
            return;
        }
        
        if (token != null) {
            tokenService.setToken(token);
            jframe.setLoggedIn(true);
        }
        
        if (remember && token != null) {
            tokenService.saveToken(token);
        }
        
        jframe.loggedSuccess(token);
    }
});
```

## GridBagConstraints Configuration

The form uses consistent constraints:

```java
GridBagConstraints gbc = new GridBagConstraints();
gbc.insets = new Insets(5, 5, 5, 5);
gbc.fill = GridBagConstraints.HORIZONTAL;
```

**Key properties**:
- `insets`: 5px spacing between components
- `fill`: HORIZONTAL - components expand to fill width
- `weightx`: 0 for labels, 1.0 for input fields (allows fields to grow)

<Tip>
The GridBagLayout allows precise control over component positioning and sizing. Labels use `weightx = 0` to maintain their natural width, while fields use `weightx = 1.0` to expand and fill available space.
</Tip>

## Security Considerations

### Password Field

The form uses `JPasswordField` instead of `JTextField`:

```java
fieldPassword = new JPasswordField(15);
```

This ensures:
- Characters are masked (displayed as dots/asterisks)
- Password is stored as `char[]` not String
- Can be cleared from memory after use

### Password Extraction

```java
String password = new String(fieldPassword.getPassword());
```

Login.java:153

<Note>
While this creates a String (which is less secure than char[]), the password is immediately sent to the authentication API and not stored long-term in the application.
</Note>

## Best Practices

<Tip>
When extending or modifying the Login panel:
- Always validate input before attempting authentication
- Provide clear, user-friendly error messages
- Never log or display the actual password
- Clear sensitive data from memory after authentication
- Use the TokenService for all token operations
- Provide visual feedback for all user actions
- Test with empty inputs, invalid credentials, and network failures
</Tip>
