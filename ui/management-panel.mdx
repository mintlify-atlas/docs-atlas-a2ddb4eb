- Updates dynamically with page changes

```java
private void updatePagination() {
    int totalItems = fullMediaList.size();
    double result = totalItems / pageSize;
    int totalPages = (int) Math.ceil(result);
    
    labelPageStatus.setText("Page " + currentPage + " of " + totalPages);
    buttonNext.setEnabled(currentPage < totalPages);
    buttonPrev.setEnabled(currentPage > 1);
    tableModel.updatePage(currentPage, pageSize);
    
    // Reset scroll to top
    SwingUtilities.invokeLater(() -> {
        tableScrollPane.getVerticalScrollBar().setValue(0);
    });
}
```

PanelManagement.java:530-551

<Accordion title="Why Math.ceil()?">
Using `Math.ceil()` ensures that any remaining items beyond a full page get their own page. For example, 105 items with pageSize=50 results in 3 pages (not 2.1).
</Accordion>

## Media Selection Helper

```java
private Media getSelectedMedia(String action) {
    int selectedRow = tableFiles.getSelectedRow();
    if (selectedRow < 0) {
        utils.showFeedback(infoLabel, 
            "Please, select a File to " + action + ".", true);
        return null;
    }
    int modelRow = tableFiles.convertRowIndexToModel(selectedRow);
    MediaTableModel model = (MediaTableModel) tableFiles.getModel();
    Media file = model.getFile(modelRow);
    return file;
}
```

## Loading State Management

### Freeze UI

```java
private void freezeUI() {
    if (isLoadingOverlayVisible) return;
    isLoadingOverlayVisible = true;
    
    SwingUtilities.invokeLater(() -> {
        JRootPane rootPane = SwingUtilities.getRootPane(PanelManagement.this);
        if (rootPane != null) {
            rootPane.setGlassPane(loadingPanel);
            loadingPanel.setVisible(true);
            loadingPanel.requestFocusInWindow();
        }
        
        Window window = SwingUtilities.getWindowAncestor(PanelManagement.this);
        if (window != null) {
            window.setCursor(Cursor.getPredefinedCursor(Cursor.WAIT_CURSOR));
        }
    });
}
```

## Real-time Synchronization

The panel uses `MediaSyncPolling` for automatic updates:

```java
private void startPolling() {
    MediaSyncPolling mediaSyncPolling = jframe.getComponent();
    token = tokenService.getToken();
    mediaSyncPolling.setToken(token);
    mediaSyncPolling.setPollingInterval(5);
    
    if (!listenerAdded) {
        mediaSyncPolling.addCustomEventListener(new CheckListMediaListener() {
            @Override
            public void checkListMediaReceived(CheckListMediaEvent evt) {
                List<Media> newMedias = evt.getMediaList();
                
                if (initialLoadInProgress && !networkReady) {
                    networkReady = true;
                }
                
                if (newMedias != null && !newMedias.isEmpty()) {
                    applyNetworkUpdateTable(newMedias);
                }
                
                if (initialLoadInProgress) {
                    checkSynchronization();
                }
            }
        });
        listenerAdded = true;
    }
    
    mediaSyncPolling.setRunning(true);
    mediaSyncPolling.setPollingInterval(60); // Change to 60s after initial load
}
```

PanelManagement.java:380-417

<Accordion title="Why two polling intervals?">
The initial polling interval is 5 seconds for quick synchronization on startup. After the initial load completes, it changes to 60 seconds to reduce server load and prevent the app from becoming unresponsive with too frequent updates.
</Accordion>

## Initialization Workflow

The panel loads data automatically when displayed:

```java
@Override
public void addNotify() {
    super.addNotify();
    
    if (!hasBeenInitialized) {
        hasBeenInitialized = true;
        initialLoad();
    } else {
        unfreezeUI();
    }
}

private void initialLoad() {
    Timer timer = new Timer(0, e -> {
        initialLoadInProgress = true;
        localReady = false;
        networkReady = false;
        
        freezeUI();
        refreshLocalList();
        refreshTableData(true);
        startPolling();
    });
    timer.setRepeats(false);
    timer.start();
}
```

PanelManagement.java:331-363

<Info>
Using a Timer with 0 delay ensures the panel paints before loading data, preventing UI freeze during initial load.
</Info>

## Best Practices

<Tip>
When working with PanelManagement:
- Always use `SwingWorker` for long-running operations to keep UI responsive
- Call `refreshLocalList()` after upload/download/delete operations
- Use the `freezeUI()`/`unfreezeUI()` pattern for operations that take time
- Validate media selection before performing actions
- Provide user feedback through the `infoLabel` for all operations
</Tip>
