## Overview

The `PanelPreferences` class (PanelPreferences.java:19) provides a comprehensive settings interface where users can configure:

- Output directory for downloads
- yt-dlp executable location

        textFieldLimDown.setText(prefService.getLimitSpeed());
        labelLimDown.setText("Speed: (Current: " +

**Validation**

<Accordion title="Why use ValidationService?">
Centralizing validation logic in a service:
- Ensures consistent validation across the application
- Makes validation rules reusable
- Simplifies testing of validation logic
- Provides clear error messages
</Accordion>

**Speed Limit Format**

- Value is stored and displayed in megabytes (MB)
- Setting to `0` means unlimited speed
- Value is applied to yt-dlp downloads via command-line arguments

## User Workflow

### Setting Download Directory

1. User clicks the folder icon button
2. File chooser opens at default location
3. User navigates to desired directory
4. User selects directory and clicks "Open"
5. PreferencesService saves the path
6. Confirmation dialog shows selected path

### Configuring Speed Limit

1. User enters numeric value in text field
2. User clicks save icon button
3. ValidationService validates the input
4. If valid:
   - PreferencesService saves the value
   - Label updates to show "(Current: XMB)"
   - Text field shows saved value
5. If invalid:
   - Error message appears in info label
   - Previous value remains unchanged

<Accordion title="What happens with invalid input?">
If the user enters non-numeric text or negative numbers:
- ValidationService throws an exception
- The catch block displays "Only integer number." error
- The previous valid value is retained
- The text field and label are not updated
</Accordion>

### Locating yt-dlp

1. User clicks the yt-dlp icon button
2. File chooser opens at "C:\\Program Files"
3. User navigates to yt-dlp.exe location
4. User selects the executable file
5. PreferencesService saves the path
6. Confirmation dialog shows selected path

<Tip>
yt-dlp must be separately installed. The application requires the path to the executable to perform YouTube downloads.
</Tip>

## Best Practices

<Tip>
When working with PanelPreferences:
- Always validate user input before saving to preferences
- Provide immediate visual feedback for all actions
- Show current values when the panel loads
- Use confirmation dialogs to confirm important changes
- Handle both valid and invalid input gracefully
- Keep the UI responsive - don't block on file operations
- Test with various input types (empty, negative, large numbers)
</Tip>
